# Example Principle Notes

这个文件用来沉淀可复用的原则层经验。

## 入口位置定义作用范围

UI 入口的位置会定义用户对对象作用范围的理解。入口放错层级，即使功能可用，心智也会出错。

例子：

- 一个用户级资产入口如果放在每个会话里，用户会自然怀疑每个会话看到的是不是不同内容。
- 一个会话级弱提示可以表达“当前正在用什么”，但不应该承担“资产归属在哪里”的职责。

## 归属先于可达

一个功能可以从某个页面快速进入，不代表它的主入口就应该属于这个页面。先判断归属，再决定是否提供上下文捷径。

## 先判断，再画界面

改原型前，先明确：

- 对象是什么
- 它属于谁
- 它应该在哪一层暴露
- 当前页面的主任务是什么

如果这些问题没有先说清，原型很容易只优化“能不能点到”，而不是“用户会怎么理解”。

## 资产态和运行态分开表达

用户可以在全局拥有某个资产，同时只在某个会话里临时使用它。UI 通常应该：

- 给资产一个全局或个人层的归属入口
- 在当前任务附近轻量表达它的运行态
